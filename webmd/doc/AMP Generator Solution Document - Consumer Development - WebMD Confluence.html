AMP Generator Solution Document

Skip to end of metadata
  • Created by Ghantasala, Mahesh, last modified on Mar 24, 2016
Go to start of metadata

1.0 Overview

The AMP Generator is a NodeJS based system that is used for generating Google AMP version of our news articles. It is made up of two components:

  1. Web Application
  2. Console Application.

 

2.0 AMP Web App

The AMP Web application takes an article chronicle id as the query parameter and generates the AMP version for the same. 

The WebApp fetches the article XML by calling Solr Search API and then parses the XML into JSON object and passes it to the EJS templating engine to generate the AMP version of the article.

3.0 Console Application

The console application is responsible for actually generating the static AMP versions of news articles. The logic is as follows:

  1. On app start it calls the new Solr Search API to fetch latest news articles since the last run. It requests for 1000 articles at a time.
    1. The Search API returns basic metadata about the articles in the result.
  2. The app iterates through the list of returned articles and for each article, makes call to the Web App passing it the chronicle id.
  3. The returned HTML is then stored in the local file system.

 

4.0 System Architecture

4.1 Components

  1. Linux box to host the AMP Generator and AMP Web App.
    1. It should have NodeJS installed.
  2. Linux box should have access to
    1.  Solr Search API.
    2. Origin Server/NAS to write the static files to
  3. Simple web server running on https://amp.<env>.webmd.com serving the static HTML files. (Or this could be akamai CNAME pulling content from origin server)
